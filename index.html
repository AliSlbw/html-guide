Referensi Elemen HTML

Panduan lengkap semua elemen HTML — fungsi, atribut, contoh kode, dan pratinjau langsung.

<html> <head> <body> <title> <meta> <link> <style> <base> Metadata & Dokumen Metadata

Elemen-elemen ini mendefinisikan struktur dokumen dan metadata halaman. Berada di dalam <head> dan tidak tampil di layar secara langsung.

Struktur Dasar Dokumen HTML
<!DOCTYPE html>
<html lang="id">
<head>
  <!-- Karakter encoding -->
  <meta charset="UTF-8"/>
  <!-- Responsif di mobile -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <!-- Deskripsi untuk SEO -->
  <meta name="description" content="Deskripsi singkat halaman"/>
  <!-- Open Graph (untuk share di sosmed) -->
  <meta property="og:title" content="Judul Halaman"/>
  <!-- Judul tab browser -->
  <title>Judul Halaman</title>
  <!-- Link ke CSS eksternal -->
  <link rel="stylesheet" href="style.css"/>
  <!-- Favicon -->
  <link rel="icon" href="favicon.ico"/>
  <!-- CSS internal -->
  <style> body { margin: 0; } </style>
  <!-- Base URL relatif -->
  <base href="https://example.com/"/>
</head>
<body>
  <!-- Konten halaman di sini -->
</body>
</html>
<div> Division / Wadah Blok Struktur

Elemen container generik bersifat block-level. Digunakan untuk mengelompokkan elemen lain agar bisa diberi style CSS atau dimanipulasi dengan JavaScript. Tidak memiliki makna semantik.

Contoh Penggunaan
<div class="kartu" id="produk-1" data-id="42">
  <h2>Judul Produk</h2>
  <p>Deskripsi produk.</p>
</div>
AtributTipeKeterangan
idstringIdentifikasi unik elemen
classstringNama kelas untuk styling CSS
stylestringCSS inline langsung di elemen
data-*stringAtribut data kustom, misal data-id="5"
hiddenbooleanMenyembunyikan elemen dari tampilan
tabindexnumberUrutan fokus keyboard
💡
Tips: Gunakan <div> untuk layout murni. Kalau kontennya punya makna khusus, pakai elemen semantik seperti <nav>, <article>, dll.
<span> Span / Wadah Inline Struktur

Mirip <div> tapi bersifat inline — tidak memulai baris baru. Berguna untuk memberi style pada sebagian teks dalam paragraf.

Contoh Penggunaan
<p>
  Harga <span style="text-decoration:line-through;color:#aaa">Rp50.000</span>
  sekarang <span style="color:red;font-weight:bold">Rp30.000</span>
</p>
<section> Section / Bagian Konten Struktur

Mengelompokkan konten tematik yang terkait, biasanya memiliki heading di dalamnya. Berbeda dengan <div>, <section> punya makna semantik.

Contoh Penggunaan
<section>
  <h2>Tentang Kami</h2>
  <p>Kami adalah perusahaan teknologi...</p>
</section>
<p> Paragraph Teks

Unit dasar teks. Browser otomatis memberi jarak atas dan bawah pada setiap paragraf.

Contoh Penggunaan
<p>Ini paragraf pertama.</p>
<p>Ini paragraf kedua, dengan spasi otomatis dari paragraf pertama.</p>
<p style="text-align:center">Teks di tengah</p>
<h1> — <h6> & <hgroup> Heading / Judul Teks

Elemen judul dengan 6 tingkatan. <hgroup> mengelompokkan judul dengan sub-judul. Penting untuk hierarki konten dan SEO.

Contoh Penggunaan
<h1>Judul Halaman Utama</h1>
<h2>Sub Judul Bagian</h2>
<h3>Sub-sub Judul</h3>
<h4>Tingkat 4</h4>
<h5>Tingkat 5</h5>
<h6>Tingkat 6 (terkecil)</h6>

<!-- hgroup: judul + sub-judul -->
<hgroup>
  <h2>Produk Terbaru</h2>
  <p>Diperbarui Maret 2025</p>
</hgroup>
⚠️
SEO: Gunakan hanya satu <h1> per halaman. Jangan melompati urutan (misal dari h1 langsung ke h4).
<blockquote> Kutipan Panjang Teks

Menandai kutipan dari sumber lain. Atribut cite menunjuk URL sumber. Berbeda dengan <q> yang untuk kutipan inline pendek.

Contoh Penggunaan
<blockquote cite="https://sumber.com/artikel">
  <p>"Belajar tanpa berpikir adalah sia-sia. Berpikir tanpa belajar
  adalah berbahaya." — Konfusius</p>
</blockquote>
<pre> & <code> Teks Preformat & Kode Teks

<pre> mempertahankan spasi/baris baru. <code> menandai cuplikan kode. Biasanya dikombinasikan untuk blok kode multi-baris.

Contoh Penggunaan
<!-- Inline code -->
<p>Gunakan fungsi <code>console.log()</code> untuk debug.</p>

<!-- Blok kode -->
<pre><code>
function sapa(nama) {
  return "Halo, " + nama;
}
</code></pre>
<ul> <ol> <li> <dl> <dt> <dd> <menu> Daftar / List Teks

<ul> = unordered (bullet), <ol> = ordered (nomor), <dl> = daftar definisi dengan <dt> (term) dan <dd> (definisi). <menu> mirip <ul> untuk daftar perintah.

Semua Jenis List
<!-- Unordered -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
</ul>

<!-- Ordered -->
<ol start="1" type="1">
  <li>Langkah pertama</li>
  <li value="5">Lompat ke 5</li>
</ol>

<!-- Definition list -->
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>

<!-- Menu (mirip ul, untuk toolbars) -->
<menu>
  <li>Simpan</li>
  <li>Cetak</li>
</menu>
<hr> & <br> Pemisah & Baris Baru Teks

<hr> membuat garis pemisah tematik (void element). <br> membuat baris baru di dalam inline text (void element). Keduanya tidak butuh tag penutup.

Contoh Penggunaan
<p>Baris pertama<br/>Baris kedua di paragraf yang sama</p>

<hr/>

<p>Konten setelah garis pemisah.</p>
<address> Informasi Kontak Semantik

Menandai informasi kontak penulis atau pemilik halaman. Browser biasanya menampilkannya dalam italic. Elemen semantik — bukan sekadar container biasa.

Contoh Penggunaan
<address>
  Ditulis oleh <a href="mailto:[email protected]">Akbar</a><br/>
  Kunjungi: <a href="https://example.com">example.com</a>
</address>
<a> Anchor / Tautan Inline

Membuat hyperlink ke halaman lain, file, email, nomor telepon, atau lokasi di halaman yang sama.

Contoh Penggunaan
<a href="https://google.com" target="_blank" rel="noopener noreferrer">Buka Google</a>
<a href="mailto:[email protected]">Kirim Email</a>
<a href="tel:+6281234">Telepon</a>
<a href="file.pdf" download="dokumen.pdf">Unduh PDF</a>
<a href="#bagian-kontak">Ke Bagian Kontak</a>
AtributTipeKeterangan
hrefURL/stringTujuan link (URL, mailto:, tel:, #id)
targetstring_blank=tab baru, _self=tab sama, _parent, _top
downloadstringPaksa file diunduh, nilai = nama file
relstringnoopener noreferrer untuk keamanan target="_blank"
hreflangstringBahasa dokumen tujuan (misal id, en)
<strong> & <em> Penting & Penekanan Semantik Inline

<strong> = konten penting (tampil tebal, punya bobot semantik). <em> = penekanan ucapan (tampil miring, punya makna semantik). Berbeda dari <b> dan <i> yang hanya visual.

Contoh Penggunaan
<p><strong>Peringatan:</strong> Jangan lupa simpan data kamu.</p>
<p>Tekan tombol <em>Simpan</em> untuk menyimpan perubahan.</p>
<!-- Kombinasi -->
<strong><em>Sangat penting!</em></strong>
<b> <i> <u> <s> <del> <ins> <small> Gaya Teks Visual Inline

Elemen inline yang mengubah tampilan teks. <b>/<i> hanya visual (tanpa makna semantik), berbeda dengan <strong>/<em>.

Contoh Penggunaan
<b>Tebal visual saja</b>
<i>Miring visual saja (istilah, judul karya)</i>
<u>Bergaris bawah</u>
<s>Dicoret (sudah tidak relevan)</s>
<del>Dihapus dari dokumen</del>
<ins>Ditambahkan ke dokumen</ins>
<small>Teks kecil, syarat & ketentuan</small>
<abbr> & <dfn> Singkatan & Definisi Inline

<abbr> menandai singkatan/akronim — atribut title menampilkan kepanjangannya saat hover. <dfn> menandai istilah yang sedang didefinisikan.

Contoh Penggunaan
<p>
  Belajar <abbr title="HyperText Markup Language">HTML</abbr> itu menyenangkan.
</p>

<p>
  <dfn>Algoritma</dfn> adalah sekumpulan instruksi terstruktur untuk menyelesaikan masalah.
</p>
<code> <kbd> <samp> <var> Elemen Teknis Inline Inline

Empat elemen khusus untuk konten teknis: kode, input keyboard, output program, dan variabel matematis/pemrograman.

Contoh Penggunaan
<!-- code: cuplikan kode -->
<p>Panggil fungsi <code>render()</code> untuk menggambar.</p>

<!-- kbd: tombol keyboard yang harus ditekan -->
<p>Tekan <kbd>Ctrl</kbd> + <kbd>S</kbd> untuk menyimpan.</p>

<!-- samp: output/hasil dari program -->
<p>Output: <samp>Error: file tidak ditemukan</samp></p>

<!-- var: variabel dalam matematika/kode -->
<p>Rumus: <var>E</var> = <var>m</var>c²</p>
<mark> Sorotan / Highlight Inline

Menandai teks yang relevan/penting dalam konteks tertentu — seperti hasil pencarian yang ditemukan. Browser menampilkannya dengan latar kuning secara default.

Contoh Penggunaan
<p>
  Hasil pencarian: "...menggunakan <mark>JavaScript</mark> untuk animasi..."
</p>
<sub> & <sup> Subscript & Superscript Inline

<sub> = teks lebih kecil di bawah baseline (rumus kimia). <sup> = teks lebih kecil di atas baseline (pangkat, catatan kaki).

Contoh Penggunaan
<!-- Rumus kimia -->
<p>Air adalah H<sub>2</sub>O</p>

<!-- Pangkat -->
<p>E = mc<sup>2</sup></p>

<!-- Catatan kaki -->
<p>Teks dengan catatan kaki<sup>[1]</sup></p>
<time> <data> <cite> Waktu, Data & Sitasi Inline

<time> menandai tanggal/waktu yang bisa dibaca mesin. <data> mengaitkan nilai mesin dengan konten manusia. <cite> menandai judul karya yang dikutip.

Contoh Penggunaan
<!-- time: tanggal bisa dibaca mesin -->
<p>Artikel diterbitkan: <time datetime="2025-03-07">7 Maret 2025</time></p>
<time datetime="14:30">Pukul 14.30</time>

<!-- data: nilai mesin untuk konten teks -->
<data value="21053">Cherry Cola</data>

<!-- cite: judul karya yang dikutip -->
<p>Seperti disebutkan dalam <cite>Clean Code</cite>.</p>
<bdi> & <bdo> Bidirectional Text Inline

<bdi> mengisolasi teks yang arahnya tidak diketahui dari konten sekitarnya (misal nama pengguna dari database). <bdo> memaksa arah teks tertentu.

Contoh Penggunaan
<!-- bdi: isolasi teks arab/RTL di dalam teks LTR -->
<p>Pengguna <bdi>مرحبا</bdi> memberi komentar.</p>

<!-- bdo: paksa arah tertentu -->
<bdo dir="rtl">Teks ini dibaca dari kanan</bdo>
<bdo dir="ltr">Left to right</bdo>
<ruby> <rt> <rp> Ruby Annotation Inline

Digunakan untuk anotasi ruby — teks kecil di atas karakter (umum di Jepang/Cina untuk pengucapan). <rt> = teks ruby, <rp> = tanda kurung fallback.

Contoh Penggunaan
<ruby>
  漢字
  <rp>(</rp>
  <rt>Kanji</rt>
  <rp>)</rp>
</ruby>
<wbr> & <q> Word Break & Kutipan Inline Inline

<wbr> menandai posisi potensial pemisah kata yang panjang. <q> untuk kutipan pendek inline — browser otomatis tambah tanda kutip.

Contoh Penggunaan
<!-- wbr: kasih peluang potong di kata panjang -->
<p>https://www.example<wbr/>.com/halaman<wbr/>/sangat/panjang</p>

<!-- q: kutipan inline -->
<p>Dia berkata <q cite="https://sumber.com">halo dunia</q>.</p>
<figure> & <figcaption> Gambar dengan Keterangan Semantik

<figure> membungkus media (gambar, diagram, kode, dll) yang direferensikan dari konten utama. <figcaption> memberi keterangan/caption.

Contoh Penggunaan
<figure>
  <img src="diagram.png" alt="Diagram Arsitektur"/>
  <figcaption>Gambar 1: Arsitektur sistem microservice</figcaption>
</figure>
<details> & <summary> Konten Lipat / Accordion Interaktif

Widget buka-tutup nativa HTML — tanpa JavaScript! <summary> adalah judul yang selalu terlihat dan bisa diklik. Tambahkan atribut open agar terbuka secara default.

Contoh Penggunaan
<details>
  <summary>Klik untuk lihat detail</summary>
  <p>Konten tersembunyi yang muncul saat diklik.</p>
</details>

<!-- Terbuka secara default -->
<details open>
  <summary>FAQ: Cara daftar?</summary>
  <p>Klik tombol Daftar di kanan atas.</p>
</details>
Pratinjau Langsung
Klik untuk lihat detail

Konten tersembunyi yang muncul saat summary diklik. Tidak perlu JavaScript!

<img> Image / Gambar Media

Menampilkan gambar. Void element — tidak ada tag penutup. Atribut alt wajib untuk aksesibilitas dan SEO. loading="lazy" untuk performa.

Contoh Penggunaan
<!-- Gambar dasar -->
<img src="foto.jpg" alt="Deskripsi gambar"/>

<!-- Dengan ukuran dan lazy load -->
<img
  src="hero.webp"
  alt="Hero banner"
  width="1200"
  height="600"
  loading="lazy"
  decoding="async"
/>

<!-- Gambar responsif dengan srcset -->
<img
  src="gambar-800.jpg"
  srcset="gambar-400.jpg 400w, gambar-800.jpg 800w"
  sizes="(max-width:600px) 400px, 800px"
  alt="Gambar responsif"
/>
AtributTipeKeterangan
srcURLPath/URL gambar — wajib
altstringTeks alternatif — wajib untuk aksesibilitas
width/heightnumberUkuran dalam piksel, mencegah layout shift
loadingstringlazy = tunda muat hingga terlihat, eager = segera
srcsetstringDaftar gambar untuk resolusi berbeda
sizesstringAturan ukuran untuk srcset
<picture> & <source> Gambar Adaptif Media

Memungkinkan browser memilih format/ukuran gambar terbaik berdasarkan kondisi. <source> mendefinisikan kondisi, <img> sebagai fallback.

Contoh Penggunaan
<picture>
  <!-- Format modern (AVIF) -->
  <source srcset="hero.avif" type="image/avif"/>
  <!-- Format WebP -->
  <source srcset="hero.webp" type="image/webp"/>
  <!-- Layar kecil vs besar -->
  <source media="(min-width:800px)" srcset="hero-wide.jpg"/>
  <!-- Fallback wajib -->
  <img src="hero.jpg" alt="Hero"/>
</picture>
<svg> Scalable Vector Graphics Media

Grafik vektor langsung di HTML — tidak pecah saat dizoom. Bisa berisi rect, circle, path, text, animasi, dan banyak lagi.

Contoh Penggunaan
<svg width="200" height="100" viewBox="0 0 200 100" xmlns="http://www.w3.org/2000/svg">
  <rect x="10" y="10" width="180" height="80" rx="8" fill="#4f8ef7"/>
  <circle cx="100" cy="50" r="25" fill="white"/>
  <text x="100" y="55" text-anchor="middle" fill="#4f8ef7" font-size="14">SVG</text>
</svg>
Pratinjau Langsung
<canvas> Kanvas Gambar Dinamis Media

Area piksel yang bisa digambar menggunakan JavaScript. Digunakan untuk game, grafik, animasi, dan visualisasi data. Konten fallback di dalamnya ditampilkan jika canvas tidak didukung.

Contoh Penggunaan
<canvas id="myCanvas" width="400" height="200">
  Browser tidak mendukung canvas.
</canvas>

<script>
  const canvas = document.getElementById('myCanvas');
  const ctx = canvas.getContext('2d');
  ctx.fillStyle = '#4f8ef7';
  ctx.fillRect(20, 20, 150, 100);
  ctx.beginPath();
  ctx.arc(300, 100, 60, 0, Math.PI * 2);
  ctx.fillStyle = '#a78bfa';
  ctx.fill();
</script>
Pratinjau Langsung
Browser tidak mendukung canvas.
<video> <source> <track> Video Embed Media

Menyematkan video langsung. Gunakan beberapa <source> untuk format berbeda. <track> untuk subtitle/caption.

Contoh Penggunaan
<video
  controls
  width="720"
  height="405"
  poster="thumbnail.jpg"
  preload="metadata"
  muted
  autoplay
  loop
>
  <source src="video.mp4"  type="video/mp4"/>
  <source src="video.webm" type="video/webm"/>
  <track
    kind="subtitles"
    src="subtitle-id.vtt"
    srclang="id"
    label="Indonesia"
    default
  />
  Browser tidak mendukung video.
</video>
<audio> Audio Embed Media

Menyematkan audio langsung di halaman. Gunakan beberapa <source> untuk kompatibilitas format lintas browser.

Contoh Penggunaan
<audio controls preload="auto">
  <source src="lagu.mp3"  type="audio/mpeg"/>
  <source src="lagu.ogg"  type="audio/ogg"/>
  <source src="lagu.webm" type="audio/webm"/>
  Browser tidak mendukung audio.
</audio>
<map> & <area> Image Map Media

Membuat area klik berbeda di dalam satu gambar. <map> mendefinisikan peta, <area> mendefinisikan setiap zona yang bisa diklik.

Contoh Penggunaan
<img src="peta.png" usemap="#petaku" alt="Peta Interaktif"/>

<map name="petaku">
  <!-- shape: rect, circle, poly -->
  <area shape="rect"   coords="0,0,100,100"     href="zona-a.html" alt="Zona A"/>
  <area shape="circle" coords="200,150,50"       href="zona-b.html" alt="Zona B"/>
  <area shape="poly"   coords="300,10,380,80,260,80" href="zona-c.html" alt="Zona C"/>
</map>
<form> Form / Formulir Form

Wadah untuk semua elemen input. Mengumpulkan data pengguna dan mengirimnya ke server. Atribut action menentukan tujuan, method menentukan cara pengiriman.

Contoh Penggunaan
<form
  action="/kirim"
  method="POST"
  enctype="multipart/form-data"
  autocomplete="on"
  novalidate
>
  ...input fields...
</form>
AtributTipeKeterangan
actionURLURL tujuan pengiriman data
methodstringGET (di URL) atau POST (di body)
enctypestringmultipart/form-data untuk upload file
autocompletestringon/off — saran pengisian otomatis
novalidatebooleanNonaktifkan validasi HTML bawaan
<input> Input / Kolom Isian Form

Elemen input serbaguna. Perilakunya berubah total sesuai type. Void element (tidak ada tag penutup). Tersedia 22+ tipe input berbeda.

Semua Tipe Input
<!-- Teks -->
<input type="text"           placeholder="Nama lengkap"/>
<input type="email"          placeholder="[email protected]"/>
<input type="password"       placeholder="Kata sandi"/>
<input type="search"         placeholder="Cari..."/>
<input type="url"            placeholder="https://..."/>
<input type="tel"            placeholder="08xx-xxxx"/>
<!-- Angka -->
<input type="number"         min="0" max="100" step="1"/>
<input type="range"          min="0" max="100" value="50"/>
<!-- Tanggal & Waktu -->
<input type="date"/>
<input type="time"/>
<input type="datetime-local"/>
<input type="month"/>
<input type="week"/>
<!-- Picker -->
<input type="color"          value="#4f8ef7"/>
<input type="file"           accept="image/*" multiple/>

Pratinjau Berbagai Input
<button> Tombol Form

Tombol interaktif yang bisa berisi HTML (teks, ikon, gambar). Lebih fleksibel dari <input type="button">.

Contoh Penggunaan
<button type="submit">Kirim Form</button>
<button type="reset">Reset</button>
<button type="button" onclick="alert('Halo!')">Klik Saya</button>
<button type="button">🔍 Cari</button>
<button disabled>Tidak Aktif</button>
<button form="form-id" formaction="/simpan">Simpan ke Server Lain</button>
<select> <option> <optgroup> Dropdown / Pilihan Form

Menu dropdown pilihan. <option> untuk setiap pilihan, <optgroup> untuk pengelompokan. Tambah atribut multiple untuk pilihan jamak.

Contoh Penggunaan
<select name="kota" required>
  <option value="" disabled selected>-- Pilih Kota --</option>
  <optgroup label="Jawa">
    <option value="jkt">Jakarta</option>
    <option value="sby" selected>Surabaya</option>
  </optgroup>
  <optgroup label="Sumatera">
    <option value="mdn">Medan</option>
  </optgroup>
</select>

<!-- Multi-select -->
<select name="hobi[]" multiple size="4">
  <option value="coding">Coding</option>
  <option value="gaming">Gaming</option>
</select>
<textarea> Area Teks Panjang Form

Input teks multi-baris. Ukuran bisa diatur pengguna (resize). Berbeda dari <input type="text">, <textarea> punya tag pembuka dan penutup.

Contoh Penggunaan
<textarea
  name="pesan"
  rows="5"
  cols="40"
  placeholder="Tulis pesan kamu..."
  maxlength="500"
  style="resize:vertical"
></textarea>
<fieldset> & <legend> Grup Field Form Form

<fieldset> mengelompokkan input yang terkait dalam satu kotak. <legend> memberi judul/caption pada grup tersebut. Membantu aksesibilitas.

Contoh Penggunaan
<fieldset>
  <legend>Informasi Pribadi</legend>
  <label for="nama">Nama:</label>
  <input type="text" id="nama" name="nama"/>
</fieldset>

<!-- Disabled group (semua input di dalamnya nonaktif) -->
<fieldset disabled>
  <legend>Premium Feature</legend>
  <input type="text" placeholder="Terkunci"/>
</fieldset>
<datalist> Saran Input Otomatis Form

Menyediakan daftar saran untuk elemen <input>. Pengguna tetap bisa mengetik bebas, tapi mendapat saran dari daftar. Hubungkan dengan list + id.

Contoh Penggunaan
<input type="text" list="browser-list" placeholder="Ketik browser..."/>

<datalist id="browser-list">
  <option value="Chrome"/>
  <option value="Firefox"/>
  <option value="Safari"/>
  <option value="Edge"/>

Atribut Global Global Attributes HTML Standard

Atribut global dapat digunakan pada semua elemen HTML tanpa terkecuali. Ini adalah fondasi penting untuk aksesibilitas, interaktivitas, dan pengelolaan konten.

💡
Tips: Atribut global seperti id, class, style, dan data-* adalah yang paling sering digunakan sehari-hari di hampir semua proyek web.

🔑 Identitas & Klasifikasi

AtributNilaiKeterangan
id string (unik) Identifier unik untuk elemen. Digunakan untuk CSS #id, JS getElementById, dan navigasi anchor #id.
class string (list) Satu atau lebih nama kelas dipisah spasi. Digunakan untuk CSS .class dan JS querySelectorAll.
style CSS inline CSS inline langsung pada elemen. Memiliki spesifisitas tinggi — gunakan dengan bijak, lebih baik pakai kelas CSS.
title string Teks tooltip yang muncul saat hover. Juga membantu aksesibilitas jika digunakan pada <abbr>.
lang BCP 47 tag Menentukan bahasa konten. Contoh: lang="id" (Indonesia), lang="en" (Inggris). Penting untuk screen reader dan SEO.
dir ltr | rtl | auto Arah penulisan teks. ltr = kiri ke kanan, rtl = kanan ke kiri (Arab, Ibrani), auto = deteksi otomatis.
translate yes | no Menentukan apakah konten boleh diterjemahkan tool seperti Google Translate. no untuk nama brand, kode, dll.

🎛️ Interaktivitas & Kontrol

AtributNilaiKeterangan
contenteditable true | false | plaintext-only Membuat elemen bisa diedit langsung oleh pengguna, seperti editor teks sederhana. plaintext-only hanya teks tanpa format.
draggable true | false Mengaktifkan drag & drop HTML5 pada elemen. Butuh event handler JS seperti ondragstart, ondrop.
hidden (boolean) | until-found Menyembunyikan elemen dari tampilan dan aksesibilitas. until-found = tersembunyi tapi bisa ditemukan oleh browser search (Ctrl+F).
inert (boolean) Menonaktifkan semua interaksi pada elemen dan turunannya — klik, fokus, pencarian. Berguna untuk modal overlay atau panel tidak aktif.
tabindex integer 0 = ikut urutan DOM, -1 = bisa difokus via JS tapi tidak via Tab, 1+ = urutan kustom (hindari nilai positif).
accesskey karakter tunggal Shortcut keyboard untuk mengaktifkan/fokus elemen. Implementasi berbeda tiap OS/browser. Contoh: accesskey="s".
spellcheck true | false Mengaktifkan/menonaktifkan pengecekan ejaan. Berguna pada contenteditable dan <textarea>.
autocapitalize off | none | on | sentences | words | characters Mengontrol auto-kapitalisasi pada keyboard mobile. sentences = awal kalimat, words = tiap kata, characters = semua huruf kapital.

📦 Data Kustom

AtributNilaiKeterangan
data-* string (bebas) Menyimpan data kustom pada elemen. Bisa dibaca via JS: element.dataset.namaKunci. Contoh: data-user-id="42" → el.dataset.userId.
slot string Menempatkan elemen ke dalam slot bernama di Shadow DOM (Web Components). Contoh: <span slot="judul">.
is string Menghubungkan elemen HTML standar ke Custom Element yang sudah terdaftar. Contoh: <button is="my-button">.

📖 Microdata & Referensi

AtributNilaiKeterangan
itemscope (boolean) Menandai elemen sebagai wadah item microdata Schema.org. Biasanya dikombinasikan dengan itemtype.
itemtype URL URL vocabulary Schema.org untuk tipe item. Contoh: itemtype="https://schema.org/Person". Berguna untuk SEO rich snippets.
itemprop string Nama properti microdata. Nilainya diambil dari konten elemen atau atribut content/src/href.
itemid URL Identifier global unik untuk item microdata, biasanya berupa URL atau URN.
itemref ID list Daftar ID elemen lain yang propertinya termasuk dalam item ini. Berguna jika properti tidak bisa diletakkan di dalam elemen itemscope.

🎨 Rendering & Performa

AtributNilaiKeterangan
part string (list) Mengekspos bagian internal Shadow DOM agar bisa di-style dari luar via CSS ::part(). Contoh: part="button icon".
exportparts string Meneruskan part dari Shadow DOM nested agar bisa diakses dari luar. Contoh: exportparts="button:my-button".
popover auto | manual Menjadikan elemen sebagai popover native HTML. auto = menutup saat klik luar, manual = hanya lewat JS. Dikontrol via popovertarget.
enterkeyhint enter | done | go | next | previous | search | send Mengubah label tombol Enter pada keyboard virtual mobile. Berguna untuk UX form di smartphone.
inputmode none | text | decimal | numeric | tel | search | email | url Menentukan tipe keyboard virtual yang muncul. Bisa digunakan pada elemen apapun, tidak hanya <input>.
nonce string (base64) Token sekali pakai untuk Content Security Policy (CSP). Digunakan pada <script> dan <style> untuk mencegah XSS.

  id="btn-hapus"
  class="btn btn-danger"
  data-item-id="42"
  data-confirm="Yakin ingin hapus?"
  title="Hapus item ini"
  tabindex="0"

  data-placeholder="Tulis sesuatu..."

  Ini popover native tanpa JavaScript!
</div>

aria-* ARIA Attributes Aksesibilitas

ARIA (Accessible Rich Internet Applications) adalah set atribut untuk meningkatkan aksesibilitas konten web — terutama untuk pengguna screen reader, keyboard navigation, dan assistive technology lainnya.

♿
Prinsip ARIA: "No ARIA is better than bad ARIA." Gunakan elemen HTML semantik terlebih dahulu. ARIA hanya digunakan saat elemen native tidak cukup untuk menyampaikan makna yang tepat.

🏷️ Role — Peran Elemen

Atribut role menentukan tujuan semantik elemen kepada assistive technology.

Nilai roleKeterangan
buttonElemen yang bisa diklik seperti tombol. Gunakan pada <div>/<span> yang berperan sebagai button.
linkElemen navigasi seperti anchor. Diumumkan sebagai link oleh screen reader.
navigationLandmark untuk area navigasi. Setara dengan <nav>.
mainKonten utama halaman. Setara dengan <main>. Hanya boleh satu per halaman.
bannerHeader situs utama. Setara dengan <header> di level halaman.
complementaryKonten pendukung konten utama. Setara dengan <aside>.
contentinfoFooter situs. Setara dengan <footer> di level halaman.
searchArea pencarian. Gunakan pada form atau section yang berisi fitur search.
formLandmark form. Hanya diumumkan screen reader jika memiliki label (aria-label/aria-labelledby).
dialogJendela dialog modal/non-modal. Butuh aria-labelledby dan manajemen fokus.
alertPesan penting yang langsung diumumkan screen reader saat muncul (live region implisit).
statusPesan status non-kritis. Diumumkan screen reader saat ada waktu luang.
logArea log yang berisi update berurutan, seperti chat atau riwayat aktivitas.
progressbarIndikator progres. Gunakan bersama aria-valuenow, aria-valuemin, aria-valuemax.
tabTab dalam komponen tablist. Selalu dalam konteks role="tablist".
tablistWadah untuk sekumpulan tab.
tabpanelPanel konten yang terkait dengan tab.
listboxDaftar pilihan yang bisa dipilih, seperti dropdown kustom.
optionPilihan di dalam listbox atau combobox.
comboboxInput dengan dropdown sugesti. Gunakan pada autocomplete kustom.
tooltipTooltip deskriptif yang muncul saat hover/fokus.
presentation / noneMenghapus semantik elemen. Berguna untuk elemen dekoratif seperti <table> yang dipakai untuk layout.
imgElemen yang merupakan gambar/ilustrasi. Butuh aria-label sebagai teks alternatif.
separatorPemisah visual/semantik antar konten. Setara dengan <hr>.

🏷️ Properti ARIA — Labeling & Deskripsi

AtributNilaiKeterangan
aria-label string Label teks langsung untuk elemen. Digunakan saat tidak ada teks label yang terlihat. Contoh: aria-label="Tutup dialog" pada tombol ×.
aria-labelledby ID list Menunjuk ID elemen lain sebagai label. Lebih kuat dari aria-label. Contoh: aria-labelledby="judul-modal".
aria-describedby ID list Menunjuk ID elemen yang berisi deskripsi tambahan. Diumumkan setelah label utama. Berguna untuk error message atau instruksi.
aria-details ID Menunjuk ke penjelasan panjang (seperti <details>). Lebih kaya dari aria-describedby.
aria-placeholder string Hint teks untuk widget input saat kosong. Setara ARIA dengan atribut placeholder.
aria-roledescription string Mengganti deskripsi role bawaan dengan teks kustom. Contoh: mengganti "slide" menjadi "kartu informasi".

🔘 State ARIA — Kondisi Dinamis

AtributNilaiKeterangan
aria-hidden true | false Menyembunyikan elemen dari accessibility tree. Berbeda dari CSS visibility:hidden. Gunakan untuk ikon dekoratif atau duplikat konten.
aria-expanded true | false Menandai apakah elemen yang dikontrol sedang terbuka/tertutup. Wajib pada tombol accordion, dropdown, tree node.
aria-selected true | false Menandai apakah item sedang dipilih. Digunakan pada tab, option, treeitem, row.
aria-checked true | false | mixed Status checkbox/radio/toggle. mixed = indeterminate state (checkbox induk yang sebagian anak tercentang).
aria-pressed true | false | mixed Status toggle button. true = sedang aktif/ditekan. Gunakan pada tombol yang memiliki dua state (aktif/tidak aktif).
aria-disabled true | false Menandai elemen nonaktif secara semantik. Berbeda dari atribut disabled — elemen masih bisa difokus dan dideteksi screen reader.
aria-readonly true | false Menandai widget input yang bisa dibaca tapi tidak bisa diubah. Berbeda dari readonly HTML — lebih luas penggunaannya.
aria-required true | false Menandai field wajib diisi. Setara semantik dengan atribut required HTML.
aria-invalid true | false | grammar | spelling Menandai nilai input tidak valid. grammar = kesalahan tata bahasa, spelling = ejaan salah. Gunakan bersama aria-errormessage.
aria-busy true | false Menandai elemen sedang memuat/diperbarui. Screen reader menunggu sebelum mengumumkan konten yang berubah.
aria-current page | step | location | date | time | true | false Menandai item yang merepresentasikan item saat ini dalam set. page = halaman aktif di navigasi, step = langkah aktif wizard.

📊 Properti Widget & Nilai

AtributNilaiKeterangan
aria-valuemin number Nilai minimum widget range (slider, progressbar, spinbutton).
aria-valuemax number Nilai maksimum widget range.
aria-valuenow number Nilai saat ini widget range. Update via JS saat nilai berubah.
aria-valuetext string Teks yang dibacakan sebagai nilai, mengganti aria-valuenow. Contoh: aria-valuetext="Senin" untuk slider hari.
aria-setsize integer Total jumlah item dalam set (list, tree, dll). Berguna untuk virtual scrolling yang tidak render semua item.
aria-posinset integer Posisi item dalam set. Digunakan bersama aria-setsize.
aria-level integer (1-6) Level hierarki heading atau tree node. Berguna untuk heading buatan yang bukan elemen <h1>–<h6>.
aria-errormessage ID Menunjuk ID elemen yang berisi pesan error. Digunakan bersama aria-invalid="true".
aria-controls ID list Menunjuk ID elemen yang dikontrol oleh elemen ini. Contoh: tombol yang mengontrol panel accordion.
aria-owns ID list Mendefinisikan relasi parent-child di luar DOM. Berguna saat struktur DOM tidak mencerminkan hubungan semantik.
aria-flowto ID list Menentukan urutan baca alternatif bagi screen reader. Berguna untuk layout multi-kolom yang kompleks.

📡 Live Regions — Update Dinamis

Live regions mengumumkan perubahan konten secara otomatis kepada pengguna screen reader.

AtributNilaiKeterangan
aria-live off | polite | assertive polite = umumkan saat screen reader idle (untuk status update), assertive = umumkan segera (untuk alert kritis, interupsi pengguna).
aria-atomic true | false true = baca seluruh region saat ada perubahan apapun. false = hanya bagian yang berubah.
aria-relevant additions | removals | text | all Jenis perubahan yang memicu pengumuman live region. Default: additions text.
Contoh Penggunaan ARIA Lengkap
<!-- Tombol toggle dengan state -->
<button
  role="button"
  aria-pressed="false"
  aria-label="Mode gelap"
  onclick="this.setAttribute('aria-pressed', this.getAttribute('aria-pressed')==='false'?'true':'false')"
>🌙</button>

<!-- Input dengan validasi -->
<label for="email">Email</label>
<input
  id="email"
  type="email"
  aria-required="true"
  aria-invalid="true"
  aria-errormessage="email-error"
/>
<span id="email-error" role="alert">Format email tidak valid</span>

<!-- Live region untuk notifikasi -->
<div
  aria-live="polite"
  aria-atomic="true"
  class="sr-only"
  id="status-msg"
></div>
<!-- JS: document.getElementById('status-msg').textContent = 'Data berhasil disimpan'; -->

<!-- Komponen tabs yang aksesibel -->
<div role="tablist" aria-label="Menu Utama">
  <button role="tab" aria-selected="true"  aria-controls="panel-1" id="tab-1">Beranda</button>
  <button role="tab" aria-selected="false" aria-controls="panel-2" id="tab-2">Profil</button>
</div>
<div role="tabpanel" id="panel-1" aria-labelledby="tab-1">Konten Beranda</div>
<div role="tabpanel" id="panel-2" aria-labelledby="tab-2" hidden>Konten Profil</div>

<!-- Slider aksesibel -->
<div
  role="slider"
  aria-valuemin="0"
  aria-valuemax="100"
  aria-valuenow="40"
  aria-valuetext="40 persen"
  aria-label="Volume"
  tabindex="0"
></div>
⚠️
Jangan gunakan ARIA untuk mengganti HTML semantik! Contoh: gunakan <button> bukan <div role="button">. Elemen native sudah otomatis punya role, state, keyboard support, dan focus management yang benar. ARIA hanya untuk widget kustom yang tidak punya padanan HTML.
